# PlayingStage

## 役割
演奏画面（メインゲーム画面）を制御する中心的なステージです。ノーツのスクロール、ヒット判定、発音処理、スコア更新、各UIパーツ（ドラムパッド、コンボ表示など）の管理を行います。

## 主要なフィールド
- `mStartTime`: 演奏開始時のUnity時間。
- `mDrumPad`: ドラムパッドのUI制御クラス。
- `mChipLight`: ノーツヒット時の発光演出制御クラス。
- `mResultTextColumn`: PERFECT/GREATなどの判定文字表示クラス。
- `mPlayingSpeed`: 演奏スピード（スクロール倍率）の管理クラス。
- `mChipDrawingList`: 描画すべきノーツのリスト。
- `mChipPlayingState`: 各ノーツの動的な状態（ヒット済みかなど）を保持する辞書。

## メソッド一覧

### OnOpen()
ステージ開始時に呼び出され、スコアデータの読み込み、各UIパーツの初期化（AddChild）、ノーツ状態の準備、入力管理の有効化を行います。

### Update()
毎フレーム呼び出され、以下の処理を行います。
1. 現在の演奏時間を更新し、描画すべきノーツリスト（`mChipDrawingList`）を更新します。
2. `UpdateChipState`により、自動演奏や見逃し（MISS）の判定を行います。
3. `CheckInput`により、ユーザーの入力をチェックし、ノーツとのヒット判定を行います。

### OnClose()
演奏終了時に呼び出され、再生中の音声を停止し、入力管理を無効化します。

### OnPlayintFinished()
曲が最後まで終了した際に呼び出され、`ResultStage`（リザルト画面）へ遷移します。

### UpdateChipState()
描画リスト内の各チップについて、以下の判定を行います。
- **見逃し（MISS）判定**: チップが判定範囲を過ぎた場合、オートプレイ設定に応じてMISS判定を発生させます。
- **自動発音処理**: 発音タイミングに達したチップの音を鳴らします（オートプレイON時、またはBGMなどの場合）。
- **自動ヒット判定**: オートプレイ設定がONの場合、判定ラインに重なった瞬間にヒット処理を行います。

### CheckInput()
ユーザーからのドラム入力を監視し、判定範囲内にあるチップと照合します。
- ヒットした場合、判定（PERFECT〜OK）を計算し、`OnChipHitted`を呼び出します。
- 該当するノーツがない空打ちの場合でも、対応するドラムパーツの発音とUI演出（パッドの沈み込み）を行います。

### OnChipHitted(Chip chip, JudgmentType judgeType, bool playSound, bool judge, bool hide, double time)
チップがヒット（または見逃し）された際の共通処理です。
- 状態をヒット済みに更新します。
- 音声の再生、UI演出（パッド発光、判定文字）、スコア加算を行います。
- 必要に応じてチップを非表示にします。

### PlayChipSound(Chip chip)
チップに関連付けられた音声を再生します。
- `SubChipId`が0の場合はデフォルトのドラム音を使用します。
- `SubChipId`がある場合は`WAVManager`を使用して特定のWAVファイルを再生します。
- BGMやSE、ギター/ベースのオート演奏音などもここで処理されます。
