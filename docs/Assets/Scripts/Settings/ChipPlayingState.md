# ChipPlayingState

## 役割
演奏中における個々のチップ（ノーツ）の動的な状態（表示中か、ヒット済みか、発音済みかなど）を管理するクラスです。

## 主要なフィールド
- `mChip`: 対象となるチップデータへの参照。
- `Visiable`: チップが画面に表示されるべき状態かどうか。
- `Hitted`: ユーザーまたはオートプレイによってヒットされたかどうか。
- `Uttered`: 音声が再生されたかどうか。

## メソッド一覧

### ChipPlayingState(Chip chip) (コンストラクタ)
チップを指定して初期状態を設定します。

### PreStatus()
チップの初期状態をセットアップします。ユーザー設定に基づき、そのチップが表示されるべき種類（`DisplayChipType`がUnknownでない）であれば`Visiable`をtrueにします。

### HitStatus()
チップがヒットされた時の状態に変更します。非表示になり、ヒット済み・発音済みフラグが立ちます。

### Clone()
現在の状態のコピーを作成します。
